# 如何保存并获取属性参数

将数据暂存到 HTTP 的 request 对象或者 Session 对象中，有三个注解

  • @RequestAttribute
    • 获取 HTTP 的请求（request）对象属性值，用来传递给控制器的参数
<%
  request.setAttribute("id", 1L);  // 设置请求参数
  request.getRequestDispatcher("./attribute/requestAttribute.do")
          .forward(request, response)  // 转发给控制器
%>

@RequestMapping("/attribute")
@Controller
public class AttributeController {

    @RequestMapping("/requestAttribute")
    public ModelAndView reqAttr(@RequestAttribute(name = "id", required = false) Long id) {
      ModelAndView mv = new ModelAndView();
      Role role = roleService.getRole(id);
      mv.addObject("role", role);
      mv.setView(new MappingJackson2JsonView())
      return mv;
    }
}

在浏览器和服务器保持联系的时候HTTP会创建一个会话对象，可以让我们在和服务器会话期间，通过它缓存一定的数据信息

  • @SessionAttribute 只能对类进行标注
    • 获取 HTTP 的会话（session）对象属性值，用来传递给控制器的参数

@Controller
@RequestMapping("/attribute")
// 配置数据模型的名称和类型，两者取或关系
@SessionAttributes(name = {"id"} , types = {Role.class})  
public class AttributeController{

  @RequestMapping("/sessionAttributes")
  public ModelAndView sessionAttrirs(Long id) {
      ModelAndView mv = new ModelAndView();
      Role role = roleService.getRole(id);
      mv.addObject("role", role); // 根据类型session将会保存角色信息
      mv.addObject("id", id); // 根据名称session将会保存id
      mv.setViewName("sessionAttribute")
      return mv;
  }
}
