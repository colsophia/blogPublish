# Spring Boot 面试题

# 1、介绍一下SpringBoot

SpringBoot是基于Spring创建的一个应用层框架，14年发布1.0，比以前的SSM，基于配置的方式效率要高很多

# 2、SpringBoot的优点

  • 独立运行（不需要单独安装一个tomcat容器，然后把项目部署到tomcat容器中），在Springboot中只要我们引入web-start会帮我们内嵌一个tomcat容器
  • 简化配置，约定大于配置的理念
  • 自动配置
  • 无代码生成xml配置
  • Actuator应用监控

# 3、SpringBoot中的核心注解

  • @SpringBootApplication注解，启动SpringBoot容器的话需要，是一个组合注解。
    • 有4个元注解
    • @ComponentScan：默认情况下会扫描当前包及其子包下面所有被@Component注解修饰的java类(Spring底层在解析配置类，会去解析@ComponentScan，读取basePackages，如果没有读取到会将当前配置类所在的包当做扫描包) ComponentScanAnnotationParser.java类（basePackages.idEmpty()）
      • 可以修改默认的包扫描路径
      @SpringBootApplication(scanBasePackages = "com.***.***")
      1
    • @SpringBootConfiguration：组合了@Configuration注解，也就是说明被SpringBootApplication所标识的java类就是一个java配置类
    • @EnableAutoConfiguration：打开自动装配的功能 -> META-INF/spring.factories文件中加载需要自动注入的java类

# 4、介绍下SpringBoot的自动装配原理

在启动SpringBoot项目的main方法的头部有一个@SpringBootApplication注解，在这个注解中组合了一个@EnableAutoConfiguration，这个注解的作用就是打开自动装配，而在这个注解中，又包含了@import注解，这个直接中引入了一个实现了importSelector接口的类型，在对应的SelectImports方法中会读取MATE/INF目录下的spring.factories文件中需要被自动装配的所有配置类，然后通过META-INF下面的spring-autoconfigure-metadata.properties文件做条件过滤，最后返回的就是需要自动装配的相关的对象

# 5、谈谈你对starter的理解

starter的作用是在MATA-INF目录下提供了一个spring.factories文件，在该文件中我们添加了一个需要注入到Spring容器中对应的配置类。
第三方的框架需要整合到SpringBoot项目中，也就是需要把第三方框架的核心组件注入到Spring容器中，提供的方式就是spring.factories文件。
Starter也可以理解为一个启动器
Starter的命名规范：官方的以spring-boot-starter-xxx命名；非官方：xxx-spring-boot-starter命名，如：mybatis-spring-boot-starter

# 6、SpringBoot项目需要单独的Web容器吗？

可以不需要，SpringBoot项目中我们会添加一个spring-boot-starter-web这个依赖，而这个依赖内嵌了Tomcat容器

# 7、介绍几个常用的starter

spring-boot-starter-web 提供SpringMVC + 内嵌了Tomcat容器
spring-boot-starter-redis 提供了Redis服务
mybatis-spring-boot-starter 提供了Mybatis

# 8、SpringBoot项目的启动方式

  • 直接通过main方法来运行
  • 打成jar 通过java -jar 的命令或打成war包，然后把这个war包放到web容器中
  • 使用Maven/Gradle插件来运行

# 9、SpringBoot中是如何解决跨域问题

  • 可以通过jsonp解决跨域问题，但是只支持get方式的请求
  • SpringBoot项目中通过WebMvcConfigure重写里面的addCorsMappings方法，在这个方法中添加允许跨域的相关请求
  • 通过@CorssOrigin

# 10、SpringBoot项目中如何使用log4j

在SpringBoot项目中默认提供了日志框架LogBack，所以我们如何需要引入log4j的话，必须先排除LogBack的依赖，然后再添加log4j的依赖，然后在src/main/resource目录下创建log4j.properties文件，然后就可以进行日志的相关的配置

# 11、bootstrap.yml文件的作用

SpringBoot中默认支持的属性文件有下面4种：
application.properties、application.xml、application.yml、application.yaml

bootstrap.yml文件：在SpringBoot中默认是不支持的，需要在SpringCloud环境下才支持，作用是在SpringBoot项目启动之前的一个父容器，该父容器可以在SpringBoot容器启动之前完成一些加载初始化的操作，比如加载配置中心的信息

# 12、SpringBoot中常用的注解

@ResetController
@Service
@Autowired
@Resource
@RequestMapping GetMapping PostMapping
@import 在xml文件中有个import标签，引入其他的配置文件，@import注解其实是import标签的替代，也可以导入第三方的配置类，而且和@Enablexxx可以实现功能模块的复用，@import注解还可以导入具体的类型对象到Spring容器中
@Indexed springBoot5之后提供的提升@ComponentScan性能的一个主键

# 13、Actuator的理解

Actuator是SpringBoot提供的四大神器之一，在单体架构中用到的非常少，但是在分布式微服务架构下就非常有用。

  • 如果有服务出现了故障，定位服务
  • 对整个系统的性能做出监控
  • 统一日志管理

# 14、如何整合ActiveMQ

对于集成SpringBoot和ActiveMQ，我们只需要添加spring-boot-starter-activemq的依赖，然后在application.properties属性文件中添加ActiveMQ的配置信息即可，不需要样板代码
